# ApplyBoard 求职申请管理平台

一个面向个人求职场景的申请流程管理 Demo，用于统一管理岗位投递进度、面试流程、关键提醒事项及简历版本信息。前端为纯静态应用，数据默认来自内置示例，并支持在浏览器中持久化保存，便于自用与演示切换。

## 在线体验

- https://applyboard.yingzhii.xyz/
- https://applyboard-ebon.vercel.app

---

## 项目简介

ApplyBoard 聚焦求职过程中信息分散、进度跟踪困难、节点提醒缺失等问题，通过轻量化工作台把申请管理、流程推进和信息组织放在一起。

平台提供看板视图与列表视图，覆盖从投递到 Offer 的主要阶段，包括：待投递、已投递、笔试中、面试中、Offer 中、已结束等。

---

## 核心功能

### 1. 求职流程管理

- 多阶段申请进度管理  
- 看板：按大阶段分列展示；列内纵向滚动，看板区域限高以突出主体  
- 列表：宽表展示，支持横向滚动查看全部字段  
- 看板与列表：在空白或非控件区域可鼠标拖动画布横向浏览（与底部横向滚动条互补）  
- 侧拉详情中可更新流程状态、编辑阶段关键时间，并支持删除当前申请

### 2. 优先处理提醒

- 截止日期与关键时间相关提醒  
- 高优先级申请提示  
- 待处理事项合集

### 3. 邮件与简历模块（演示）

- 邮件区：模拟面试 / 笔试等通知样式（占位能力）  
- 简历区：多版本简历与使用频次等展示（占位能力）

### 4. 真实岗位链接

部分示例记录带真实招聘链接，可在新标签页打开。

### 5. 本地数据与演示恢复

- 申请列表会写入浏览器 `localStorage`（`applyboard.applications.v1`），刷新后仍保留你的新增、改状态、删除等操作，便于自用。  
- 数据按域名与浏览器隔离：他人访问同一部署地址时，只会看到各自浏览器中的数据；首次访问若无存档，则加载内置示例数据。  
- 页面底部提供低调的「恢复示例数据」：一键还原为内置示例列表，并覆盖当前本地存档（适合演示前重置）。

## 本地运行

```bash
cd frontend
npm install
npm run dev
```

生产构建：

```bash
cd frontend
npm run build
```

---

## 设计思路

- 信息架构与 Master-Detail（列表 / 看板 + 侧拉详情）  
- 轻量化界面与可读性优先的表格 / 看板布局  
- 在无后端的前提下，用本地存储平衡「演示还原」与「真实自用」
